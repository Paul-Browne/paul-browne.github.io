Grid System

overview

The grid system is the foundation of the layout. Curator uses epicGrid to handle this. Unlike most grid systems that are written in CSS, epicGrid is written in pure javascript, which allows greater choice of grid style, more breakpoints and auto clearing of parallel columns - no need to add clearfix containers, which in turn allows for more layout variation between breakpoints.

Column types Description
prefix-col-x-y A column for the chosen viewport. X number of columns wide from a total of Y
prefix-center-x-y Center the column for that viewport. X number of columns wide from a total of Y
prefix-offset-x-y Offset the column to the right by X number of columns from a total of Y
prefix-left-x-y Displace a column to the left by X number of columns from a total of Y
prefix-right-x-y Displace a column to the right by X number of columns from a total of Y
  • choose one of the prefixes x, s, m, l or h for that viewport, obvoiusly you can use as many as you like, per element.
  • classes left, right and offset must be accompanied with a col class to specify width.
  • the class center cannot be used with left, right or offset.
breakpoints

You can use up to 5 different prefixes to independently control the layout for different sizes. Column widths are inherited and the default size is 100%. Meaning that, for example;

<div class="m-center-3-4">

This column will be 100% wide on screens less than or equal to 656px wide - since no x- or s- classes are present.
Also, this column will be three quarters wide on all screens greater than 656px wide, since no l- or h- classes are present.

size prefix breakpoint
eXtra small x- greater than 0
Small s- greater than 320px
Medium m- greater than 656px
Large l- greater than 1024px
Huge h- greater than 1440px
parallel columns

In this simple example there are 2 columns - a main column and a sidebar, three quarters and one quarter wide, respectively.
Note how there is no need to wrap a clearfix container around these columns in order to clear the floats. This is done automatically by epicGrid every time the column amount equals one.

<div id="main" class="m-col-3-4" >...</div>
<div id="sidebar" class="m-col-1-4" >...</div>
centered columns

In this simple example there are two centered columns. This example shows how versatile the grid system is by allowing any combination of "fractions" for the column sizes.

<div class="m-center-71-95" >...</div>
<div class="m-center-31-43" >...</div>
offset columns

Offsetting columns simply pushes a column to the right by that amount.
Note how the amount total of the column widths plus the offset equals one.

<div class="m-col-1-5" >...</div>
<div class="m-col-1-2 m-offset-1-10" >...</div>
<div class="m-col-1-5" >...</div>
displaces columns

Displaced columns are pushed left or right, so that content in the parent column (usually text) can flow around them.
In the below example the main (pink) column is eight tenths wide and centered, meaning that it has one tenth of "free space" on either side.
The blue columns are aligned to the left/right margin of their parent by the addition of the class m-left-0-1 or m-right-0-1.
The orange columns are pushed through the left/right margin of their parent by one tenth, because they have the class m-left-1-10 or m-right-1-10.
Note: If you intend to push a column through its parent's margin be sure that the parent has enough free space on that side. In this example the maximum amount of free space was infact one tenth.

<div class="m-center-8-10" >
    <p>...</p>
    <div class="m-col-3-5 m-left-0-1" >...</div>
    <p>...</p>
    <div class="m-col-3-5 m-left-1-10" >...</div>
    <p>...</p>
    <div class="m-col-3-5 m-right-0-1" >...</div>
    <p>...</p>
    <div class="m-col-3-5 m-right-1-10" >...</div>
    <p>...</p>
</div>

Image Tools

responsive images

Responsive images - images that change depending on container size and device pixel density, are easy to add. The example below shows 5 different images that will be loaded depending on the afore mentioned conditions.
The placeholder image should be approx. 1024x768, low quality ~20kb in size. It is loaded regardless (before being replaced), but is entierly optional.
You can specify as few, or as many breakpoints as you like, but it is highly advisable to include at least one "less than X" and one "greater than X" breakpoint, this way you gaurentee that the placeholder image is always replaced.

<img src="i/world-placehoder.jpg" class="foo" id="bar" alt=""
data-src="<320:i/world-small.jpg,
          <600:i/world-medium.jpg,
          <1024:i/world-large.jpg,
          <1440:i/world-huge.jpg,
          >1440:i/world-massive.jpg" />
image cover

Adding the class icovr to a container of an <img> will gaurentee that the image covers the container and is centered. In the same way that background-size:cover; background-position:center center; works for background images. A height for the container must be specified in order for it to work properly.

<div class="icovr" id="bar">
    <img src="i/world-medium.jpg" alt="" />
    </div>
